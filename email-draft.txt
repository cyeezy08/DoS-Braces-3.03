Subject: [Security Advisory] Denial of Service (uncontrolled resource consumption) in braces@3.0.3

Dear Jon and Brian,

I am writing to report a denial of service (DoS) vulnerability in the latest release of the `braces` package (v3.0.3). This is an incomplete fix for CVE-2024-4068.

SUMMARY
=======
While the fix for CVE-2024-4068 successfully limited the maximum input character length to 10,000, it did not place any limit on the combinatorial output size of comma-separated expansions. A malicious input as short as 110 characters can cause 4.2 million array items to be materialized in memory, consuming ~1.2GB of RAM and blocking the CPU/event-loop for up to 30 seconds.

ROOT CAUSE
==========
In `lib/expand.js`, the `rangeLimit` check only applies when `node.ranges > 0` (numeric/alphabetic ranges). Comma-separated expansions (e.g. `{a,b}`) have `node.ranges === 0` and bypass this guard entirely. The recursive `append` function continues to build combinations unchecked, leading to combinatorial explosion.

REPRODUCTION
============
```javascript
const braces = require('braces');
const input = '{a,b}'.repeat(22); // 110 characters
console.time('DoS Expand');
const result = braces.expand(input);
console.timeEnd('DoS Expand');
console.log('Result items:', result.length.toLocaleString());
// Output: 4,194,304 items in ~10-30 seconds
```

TEST DETAILS (on braces@3.0.3, item counts are deterministic)
=============================================================
Input (chars) | Output Items | Time (approx) | Memory (approx)
--------------|--------------|----------------|----------------
50            | 1,024        | ~10ms          | ~1MB

110           | 4,194,304    | ~10-30s        | ~1.2GB+

IMPACT
======
Since `braces` is a core dependency of `micromatch` (used by webpack, vitest, jest, eslint, etc.), any application or build tool processing untrusted user input via brace expansion is vulnerable to CPU and memory exhaustion DoS.

PROPOSED PATCH
==============
See the included `patch.diff` file for a complete implementation that wraps `append()` calls with output-size validation. This passes all 894 existing unit tests.
