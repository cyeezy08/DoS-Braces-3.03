I am writing to report a denial of service (DoS) vulnerability in the latest release of the `braces` package (v3.0.3). This is an incomplete fix for CVE-2024-4068.

SUMMARY
=======
While the fix for CVE-2024-4068 successfully limited the maximum input character length to 10,000, it did not place any limit on the combinatorial output size of comma-separated expansions. A malicious input as short as 110 characters can cause 4.2 million array items to be materialized in memory, consuming ~1.2GB of RAM and blocking the CPU/event-loop for nearly 30 seconds.

ROOT CAUSE
==========
In `lib/expand.js`, the `rangeLimit` check only applies when `node.ranges > 0` (numeric/alphabetic ranges). Comma-separated expansions (e.g. `{a,b}`) have `node.ranges === 0` and bypass this guard entirely. The recursive `append` function continues to build combinations unchecked, leading to combinatorial explosion.

REPRODUCTION
============
Below is a simple reproduction script:

```javascript
const braces = require('braces');
const input = '{a,b}'.repeat(22); // 110 characters
console.time('DoS Expand');
const result = braces.expand(input);
console.timeEnd('DoS Expand');
console.log('Result items:', result.length.toLocaleString());
// Output on Node 24:
// DoS Expand: ~29.5 seconds
// Result items: 4,194,304
```

IMPACT
======
Since `braces` is a core dependency of `micromatch` (used by webpack, vitest, jest, eslint, etc.), any application or build tool processing untrusted user input via brace expansion is vulnerable to CPU and memory exhaustion DoS.

PROPOSED PATCH
==============
To resolve this comprehensively, we can intercept and check the size of the combined output array *before* each concatenation step in `lib/expand.js`. This prevents combinations of consecutive braces (e.g. `{a,b}{c,d}...`) and mixtures of ranges/commas from exploding.

We can define a wrapper helper inside the `expand` function in `lib/expand.js`:

```javascript
const queueLimit = (queue, stash, enclose) => {
  if (rangeLimit === Infinity) return append(queue, stash, enclose);
  const queueLength = queue ? [].concat(queue).length : 0;
  const stashLength = [].concat(stash).length;
  const nextLength = queueLength === 0 ? stashLength : (stashLength === 0 ? queueLength : queueLength * stashLength);
  if (nextLength > rangeLimit) {
    throw new RangeError('expanded array length exceeds range limit. Use options.rangeLimit to increase or disable the limit.');
  }
  return append(queue, stash, enclose);
};
```

Replacing calls to `append(...)` inside `walk(...)` with `queueLimit(...)` fully mitigates the vulnerability while passing all 894 existing unit tests.
